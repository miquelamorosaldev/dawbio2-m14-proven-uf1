When it comes to data analysis, choosing the right tool for the job is essential. Pandas is a popular Python library for data manipulation and analysis, but it can sometimes be slow when working with large datasets. Polars is a newer library that provides a fast and efficient alternative to Pandas, with a similar API.

We will compare the syntax and performance of Pandas and Polars by creating, manipulating, and analyzing large datasets.

Importing Libraries

In this blog post, we will compare the performance of two popular data analysis libraries — Pandas and Polars. Pandas is a widely used library for data manipulation and analysis in Python, while Polars is a relatively new library that aims to provide similar functionality but with improved performance. We will first import the required libraries and then compare the syntax of various operations in Pandas and Polars.

Finally, we will compare the performance of these libraries for different operations and provide theoretical and probabilistic explanations for our observations.

First, let’s import the necessary libraries for our analysis. We will be using Pandas version 2.0.0 and Polars version 0.17.7.

Syntax Comparison

Let’s start with the Syntax Comparison by creating a DataFrame using both libraries.

Creating a DataFrame

To create a DataFrame in Pandas and Polars, we can use the DataFrame() constructor. We will create a DataFrame with randomly generated data for the purpose of comparison.

The Pandas DataFrame will have the following columns: name, age, city, income, gender, is_married , children, zip_code , interest_rate , and is_default . The Polars DataFrame will have the same columns and data types as the Pandas DataFrame. 

֫def create_dataframe(n_rows, library):
  if library == 'pandas':
    data = {
      'name': np.random.choice(['Alice', 'Bob', 'Charlie', 'David', 'Eva'], n_rows),

Saving and Reading a DataFrame

To save a DataFrame to a CSV file, we can use the to_csv() method in Pandas and the write_csv() method in Polars. To read a DataFrame from a CSV file, we can use the read_csv() function in Pandas and the read_csv() method in Polars. 

֫def save_dataframe_to_csv(df, file_name):
  """
  Saves a Pandas or Polars DataFrame to a CSV file.
  :param df: The DataFrame to save.
  :param file_name: The name of the file to save to.
  """
  if isinstance(df, pd.DataFrame):
    df.to_csv(file_name)
  elif isinstance(df, pl.DataFrame):
    df.write_csv(file_name)

def read_dataframe_from_csv(file_name, library):
  """
  Reads a Pandas or Polars DataFrame from a CSV file.
  :param file_name: The name of the CSV file to read from.
  :param library: The library to use - 'pandas' or 'polars'.
  
  :return: The DataFrame read from the CSV file.
  """
  if library == 'pandas':
    return pd.read_csv(file_name)
  elif library == 'polars':
    return pl.read_csv(file_name)

Sorting a DataFrame

To sort a DataFrame in Pandas, we can use the sort_values() method. In Polars, we can use the sort() method. Both methods take the column name to sort by and a Boolean flag to specify whether to sort in ascending or descending order. 

def sort_dataframe(df, column_name, ascending=True):
  """
  Sorts a Pandas or Polars DataFrame by the values in a given column.
  :param df: The DataFrame to sort.
  :param column_name: The name of the column to sort by.
  :param ascending: Whether to sort in ascending or descending order. Default is ascending.
  
  :return: The sorted DataFrame.
  """
  if isinstance(df, pd.DataFrame):
    return df.sort_values(by=column_name, ascending=ascending)
  elif isinstance(df, pl.DataFrame):
    return df.sort(column_name, descending=not ascending)

Filtering a DataFrame

To filter a DataFrame in Pandas, we can use boolean indexing by specifying a condition in square brackets. In Polars, we can use the filter() method with a col() function to specify the column and a condition in parentheses. 

def filter_dataframe(df, column, value):
  """
  Filters a Pandas or Polars DataFrame based on a given condition.
  :param df: The DataFrame to filter.
  :param column: The column to filter on.
  :param value: The value to filter on.
  :return: The filtered DataFrame.
  """
  if isinstance(df, pd.DataFrame):
    return df[df[column] == value]
  elif isinstance(df, pl.DataFrame):
    return df.filter(pl.col(column) == value)

Grouping a DataFrame

We can use the method to group a DataFrame in Pandas, which creates a groupby object. In Polars, we can also use the groupby() method, which returns a groupby object. We can then perform aggregate operations on the groupby object, such as count() , mean() , and sum() . 

def groupby_dataframe(df, column_name):
  """
  Groups a Pandas or Polars DataFrame by a given column name.
  :param df: The DataFrame to group.
  :param column_name: The name of the column to group by.
  :return: The grouped DataFrame.
  """
  start_time = time.time()
  result = df.groupby(column_name)
  end_time = time.time()
  return end_time - start_time

Performance Comparison

We can use the functions we defined earlier to compare the performance of Pandas and Polars. We’ll start by generating DataFrames with a million rows using both libraries and then time each operation for both libraries.

file_name_pan = 'data_pandas.csv'

# Creating DataFrames

We have used the functions create_dataframe , save_dataframe_to_csv , read_dataframe_from_csv , sort_dataframe , filter_dataframe , and groupby_dataframe to perform the operations on the DataFrames generated by both libraries. The runtimes of each operation have been stored in two dictionaries, pandas_times and polars_times .

Now we can use the compare_runtimes function to plot a bar graph comparing the runtimes of each operation for Pandas and Polars. Here is the code: 

֫def compare_runtimes(pandas_times, polars_times):
  """
  Compares runtimes of Pandas and Polars for various operations.
  :param pandas_times: A dictionary containing the runtimes for various operations for Pandas.
  :param polars_times: A dictionary containing the runtimes for various operations for Polars.
  """
  fig, ax = plt.subplots(figsize=(12, 4))

  # Setting the title and labels for the graph
  ax.set_title("Comparison of Runtimes for Pandas and Polars")
  ax.set_xlabel("Operations")
  ax.set_ylabel("Time (Seconds)")

  # Creating a bar graph for Pandas and Polars runtimes
  operations = pandas_times.keys()
  num_operations = len(operations)
  bar_width = 0.35
  pandas_x = np.arange(num_operations)
  polars_x = pandas_x + bar_width

  pandas_y = list(pandas_times.values())
  polars_y = list(polars_times.values())

  pandas_bar = ax.bar(pandas_x, pandas_y, bar_width, color='tab:blue', label='Pandas')
  polars_bar = ax.bar(polars_x, polars_y, bar_width, color='tab:orange', label='Polars')

  # Setting the x-axis ticks and labels
  ax.set_xticks(pandas_x + bar_width / 2)
  ax.set_xticklabels(operations)

  # Setting the y-axis limit as the maximum of the two maximum values of runtimes
  ax.set_ylim(0, max(max(pandas_y), max(polars_y)) * 1.2)

  # Adding values on top of the bars
  for i, bars in enumerate(zip(pandas_bar, polars_bar)):
    for bar in bars:
      height = bar.get_height()
      ax.annotate(f"{height:.4f}", xy=(bar.get_x() + bar.get_width() / 2, height),
                  xytext=(0, 3), textcoords="offset points", ha='center', va='bottom')

  # Adding a legend to the graph
  ax.legend()

  # Displaying the graph
  plt.show()
compare_runtimes(pandas_times, polars_times)

It is clear from the graph that Polars perform faster than Pandas for most operations. This is particularly true for the GroupBy operation, where Polars is nearly 20 times faster than Pandas. The Filter operation is also significantly faster in Polars, while the Create and Read operations are somewhat faster in Pandas. Overall, Polars seems to be a more performant library for data manipulation, particularly for large datasets.

The x-axis shows the different operations we performed, and the y-axis shows the time taken in seconds. The blue bars represent the runtimes for Pandas, while the orange bars represent the runtimes for Polars. The graph shows that Polars consistently perform better than Pandas in most operations. Although there are some cases where Pandas perform slightly better than Polars, the overall trend favors Polars.

It is important to note that these results are specific to the dataset and operations used in this example. Different datasets and operations may yield different results, and it is always recommended to benchmark different libraries for specific use cases to determine the optimal choice for performance.

Theoretical and Probabilistic Explanations

The performance differences between Pandas and Polars can be attributed to using different underlying data structures. Pandas uses a DataFrame, which is based on a 2-dimensional numpy array, while Polars uses a DataFrame based on Rust’s Arrow data format. This difference can explain why Polars is generally faster than Pandas, as Rust’s Arrow is designed to be more efficient for data manipulation tasks.

Another factor that can affect the performance of data manipulation libraries is the probabilistic time complexity of their algorithms. Polars has been optimized to use algorithms with lower time complexity, which means that they perform better for large datasets. On the other hand, Pandas uses algorithms with higher time complexity, making it slower for large datasets.

Conclusion

In this blog post, we compared the performance of Pandas and Polars for various data manipulation operations. We found that Polars was generally faster than Pandas, particularly for the GroupBy and Filter operations. However, Pandas was faster than Polars for the Create operation.

The performance differences can be attributed to the fact that Pandas and Polars use different underlying data structures and the probabilistic time complexity of their algorithms.

Overall, if you are working with large datasets and need to perform complex data manipulation tasks, Polars might be a more performant library to use.

However, if you work with smaller datasets, Pandas might be a more convenient option as it has a more user-friendly API.
